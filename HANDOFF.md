## 1. 최우선 버그 3건 (기능 개발보다 먼저)

### 버그 1 — 구글 리뷰 이벤트(추가이벤트) 게임 화면 깨짐
**증상**: 추가이벤트 중 구글 리뷰 게임 참여 시 소리는 나오는데 화면이 이상하게 보이고, 결과값은 나오긴 함.
**의심 원인**: `openExtraRoulette()` 함수에서 게임 뷰 전환 시(`extra-view-roulette` / `extra-view-card` / `extra-view-scratch`) 이전 게임의 잔여 스타일(transform, display)이 완전히 초기화되지 않는 것으로 추정. 또는 캔버스 크기(width/height 속성 vs CSS 크기) 불일치로 그림이 깨지는 경우도 있음.
**확인 방법**: 매장 관리자 → 추가이벤트 탭 → 구글 리뷰 ON, 게임방식을 각각 룰렛/카드/복권으로 바꿔가며 고객 화면에서 직접 참여해보고 콘솔 에러(F12) 확인.
**참고 함수**: `openExtraRoulette`, `drawExtraWheel`, `finishExtraGame`, `showExtraResult` (index.html 내 검색)

### 버그 2 — 카드뽑기 카드 크기 너무 작음
**증상**: 카드가 작아서 상품명이 다 안 보이고 "---"처럼 깨져 보임.
**해결 방향**: 카드 가로/세로 크기를 최소 20~30% 키우고, 폰트 크기도 비례해서 키울 것. 5장이 한 줄에 안 들어가면 2줄 배치(3+2)로 바꾸는 것도 고려.
**참고 함수**: `initMainCard`(메인 게임), `initExtraCard`(추가이벤트) — CSS 인라인 스타일의 `width:56px;height:80px` 부분 등

### 버그 3 (확인 필요) — 세션 중 3번 발생했던 문법 에러 패턴
과거 세션에서 "함수 중복 선언"(`function toggleEvent(ei){function toggleEvent(ei){`) 같은 실수가 있었음. 클로드 코드는 실제 실행 환경이 있으니, 코드 수정 후 반드시 브라우저에서 열어 콘솔 에러 없는지 확인하는 습관을 들일 것.

## 2. 이번 주 신규 기능 요청 (우선순위 순)

### 2-1. 쿠폰 탭 ↔ 고객·연락처 탭 통합 + 기간별 CRM 발송
**요구사항**:
- 쿠폰 탭과 고객·연락처 탭을 하나로 합친다 (고객별 최근 방문일 + 보유 쿠폰이 한 화면에)
- 고객 목록에서 개별 선택 또는 체크박스로 일괄 선택 가능
- **기간 필터**: "최근 N일 미방문 고객" 필터 (예: 3주 이상 미방문). 일별/주별/월별 프리셋 버튼 제공
- 고객명을 누르면 그 고객이 보유한 쿠폰 목록이 뜨고, 그중 어떤 쿠폰을 보낼지 선택
- 선택한 쿠폰으로 "문구 복사" 또는 "문자 보내기(`sms:` 링크)" 실행
- **기본 문구 템플릿을 매장설정에서 사장님이 직접 수정** 가능하게 (변수: {고객명}, {매장명}, {쿠폰명}, {유효기간} 등 치환)
- 문자 바이트 계산 표시: 일반 SMS 80byte, MMS(사진 3장 포함) 2000byte 기준으로 안내 문구 아래에 "현재 OO byte / 80byte(SMS)" 같은 실시간 카운터
- 발송 완료 시 날짜 기록 (이미 `sentAt` 필드로 구현되어 있음 — `markCouponSent` 함수 참고, 이번엔 UI 통합만 하면 됨)

**기존 구현 참고**: `renderAdmin` 함수의 `tab==='coupons'` 분기, `copyCouponMessage`, `markCouponSent` 함수가 이미 있음. `tab==='customers'` 분기도 있음. 이 둘을 병합하는 작업.

### 2-2. 설문 응답 — 통계 그래프로 표시
**요구사항**: 지금은 응답을 리스트로 나열만 함(각 고객 이름+날짜+답변 텍스트). 이걸 항목별로 집계해서 그래프(막대/평균 별점 등)로 보여줄 것. 특히 매트릭스형 문항(가격대비맛, 가격대비양, 조리대기시간, 매장청결위생, 직원친절도)은 항목마다 평균 별점을 막대그래프로.
**참고**: `renderSurveyAdmin` 함수 내 응답 결과(`STORE.surveys`) 렌더링 부분.

### 2-3. 홈 화면 문구/구조 변경
- "룰렛 이벤트" → **"리뷰이벤트 게임"**으로 명칭 변경 (게임 종류에 상관없이 통일된 명칭)
- 그 아래 한 칸에 "추가 이벤트 참여하기" 배너를 별도로 배치해서 카카오맵/구글/티맵 참여 유도를 더 눈에 띄게 할 것 (현재는 결과 화면에서만 뜨는데, 홈에서도 진입 가능하게)
**참고**: `s-home` 화면 HTML, `refreshHome` 함수

## 3. 지금까지 구축된 기능 전체 목록 (신규 세션이 헷갈리지 않도록)

### 고객 화면
- 전화번호+닉네임 로그인 (전화번호 기준으로 히스토리 매칭, 이름 변경 이력은 `nameHistory`에 보존)
- 메인 게임 3종: 룰렛 / 카드뽑기 / 긁는복권 (매장 관리자가 택1, 확률·상품 공유)
- 추가 이벤트 3종(카카오맵/구글/티맵): 각각 독립적으로 게임방식·상품·확률·링크 설정 가능
- VIP 등급 5단계(브론즈~다이아, 10/20/30/50회 방문 기준) + 자동 쿠폰 발급
- 스탬프 카드(고객별 독립 카운트, 목표 개수·보상 사장님이 설정, 보상 이미지 업로드 가능)
- 만족도 설문: 별점/매트릭스(여러 항목 한번에 별점)/복수선택 체크박스/객관식/주관식 5가지 문항 유형. 기본 프리셋은 5항목 매트릭스(가격대비맛/양/대기시간/청결/친절도)
- 쿠폰함, 카카오채널 당첨 공유(클립보드 복사 + 채팅 열기)

### 매장 관리자 화면 (탭 구조)
- 통계: 일/주/월 참여 차트, VIP 등급 분포, 추가이벤트 참여현황
- 고객·연락처: CSV/TXT 다운로드
- QR코드
- 룰렛(→게임방식 선택 포함)
- 쿠폰: CRM 발송 버튼(복사/문자), 발송완료 기록 ← **2-1에서 고객탭과 통합 예정**
- 설문: 문항 관리(5가지 유형), 응답 결과 ← **2-2에서 그래프화 예정**
- 추가이벤트: 플랫폼별 ON/OFF, 링크, 게임방식, 상품·확률
- 매장설정: 매장정보, 배경이미지, 네이버연동, 카카오채널/채팅 URL, 사장님 PIN, 스탬프 설정, VIP 혜택 설정

### 플랫폼 총괄관리자 화면
- 전체 매장 신호등(🟢7일이내/🟠8~14일/🔴15일+) + 휴면순 정렬
- 전체 매장 DB CSV 다운로드 (영업자료용)
- 매장별 "세팅 관리" 진입 (같은 브라우저 내에서 대신 관리 — 백엔드 없이는 다른 기기 반영 안 됨, 아래 참고)

### 데이터 저장 방식 (반드시 인지할 것)
**전부 `localStorage` 기반 데모입니다.** 브라우저별로 따로 저장되고, 다른 기기·다른 사장님 폰에서는 서로의 데이터가 안 보입니다. 이건 임시 상태이고, 실제 여러 매장이 쓰려면 공유 백엔드(Supabase 추천)로 이전해야 함 — 아래 4번 참고.

## 4. 다음 큰 단계 — Supabase 백엔드 이전 (신규 기능 다 끝난 후)

지금 구조로는 안 되는 것들(백엔드가 있어야 성립):
- 사장님이 다른 기기에서 로그인해도 같은 데이터 보기
- 총괄관리자가 매장 세팅을 "진짜로" 대신 바꿔주기 (지금은 같은 브라우저에서만 임시로 가능)
- mid 기반 하위도메인 QR을 스캔한 손님이 실제 그 매장 데이터에 연결되기
- 카카오 알림톡 자동 발송 (현재는 클립보드 복사 방식으로 대체 중)

**이전 순서 제안**:
1. Supabase 프로젝트 생성, 테이블 스키마 설계 (stores, users, coupons, spins, surveys, staff, events)
2. RLS(Row Level Security) 정책 작성 — 매장은 자기 데이터만 쓰기, 총괄은 전체 읽기
3. `storesDB`, `STORE`, `saveState()` 등 localStorage 관련 코드를 Supabase 클라이언트 호출로 순차 교체
4. 파일을 여러 개로 분리하는 것도 이 시점에 함께 고려 (index.html이 계속 커지고 있음)

## 5. 마케팅 자동화 도구(별도 프로젝트) — 참고만

대표님이 별도로 구상 중인 "구글 워크스페이스 연동 마케팅 도구"는 **이 리뷰이벤트 앱과 완전히 분리된 새 프로젝트로 시작해야 합니다.** 이유: 리뷰이벤트 앱은 고객이 만지는 공개 웹앱이고, 마케팅 도구는 대표님 팀 전용 내부 도구라 권한·보안 구조가 다릅니다. 같은 코드에 합치면 안 됩니다.

실현 가능성 정리 (참고용):
- ✅ 가능: 신규 고객가입/쿠폰발급 로그를 구글시트로 자동 전송 (Google Apps Script 웹훅, Supabase Edge Function과 연동)
- ⚠️ 조건부: 네이버 Search Ads API, DataLab API는 공식 API 있음 → 활용 가능. **단, 네이버 스마트플레이스(리뷰수·평점·방문통계) 자체는 공식 API가 없어서 자동 수집이 불가능함.** 크롤링 우회는 이용약관 위반 소지가 있어 권장하지 않음.
- ❌ 불가(구조적으로): "리뷰이벤트 앱 안에 마케팅 도구를 서브앱으로 구동"은 두 앱이 같은 데이터베이스(Supabase)를 봐야 의미가 있음. 이것도 리뷰이벤트 앱의 Supabase 이전이 먼저 끝나야 함.

이 마케팅 도구는 리뷰이벤트 앱의 Supabase 이전이 끝난 뒤, 별도 새 세션/새 프로젝트로 다시 논의하는 것을 추천.
